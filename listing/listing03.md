Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
печать самой err выдаст nil, но при этом второй вывод (err == nil) покажет false, это объясняется устройством интерфейсов в golang, они представляют собой структуру (iface), которая содержит метаданные - так называемый itable, позволяющие определить тип, а также указатель на данные, в данном случае, этот указатель у err будет равен nil (что подтверждает первый вывод), но тип интерфейса - *os.PathError (уже не nil, поэтому преобразованный  в error интерфейс уже не станет равным nil)

Пустые интерфейсы - особые интерфейсы в go, не имеющие никаких методов, поэтому их реализуют все типы данных. С точки зрения реализации, пустым интерфейсам не нужен itab, поэтому для описания пустого интерфейса используется структура eface проще, чем iface
```
