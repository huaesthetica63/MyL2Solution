Что выведет программа? Объяснить вывод программы.

```go
package main

type customError struct {
	msg string
}

func (e *customError) Error() string {
	return e.msg
}

func test() *customError {
	{
		// do something
	}
	return nil
}

func main() {
	var err error
	err = test()
	if err != nil {
		println("error")
		return
	}
	println("ok")
}
```

Ответ:
```
Будет выведена строка "error"
Это объясняется тем, как мы объявили переменную err - как переменную-интерфейс error. Интерфейсы в go имеют свои особенности реализации: они представляют собой структуру, которая содержит указатель на объект и его тип, только когда оба этих поля - nil, весь интерфейс равен nil,результат работы функции test преобразовался к интерфейсу error, указатель равен nil, а тип - *customError, поэтому сам интерфейс уже не nil, соответственно, будет выполнен if с условием err != nil

Мы можем получить иной результат, если не укажем явно тип err как error
err:=test или var err *customError

Результат объясняется только особенностями работы с интерфейсом, можно было проиллюстрировать такую же ситуацию не только с error-интерфейсом, но и с любым другим пользовательским интерфейсом и структурой, которая его реализует
```
