Что выведет программа? Объяснить вывод программы. Рассказать про внутреннее устройство слайсов и что происходит при передачи их в качестве аргументов функции.

```go
package main

import (
	"fmt"
)

func main() {
	var s = []string{"1", "2", "3"}
	modifySlice(s)
	fmt.Println(s)
}

func modifySlice(i []string) {
	i[0] = "3"
	i = append(i, "4")
	i[1] = "5"
	i = append(i, "6")
}
```

Ответ:
```
Будет выведен слайс "3" "2" "3"
Мы передали в функцию modifySlice слайс по значению, а не по указателю, поэтому операции append изменят копию s внутри функции modifySlice, и все, что будет после первого append (когда добавили элемент "4"), никак не повлияет на значение s в main, однако, перед append мы изменили 1-вый элемент в слайсе, мы к нему обратились по указателю, поэтому поменяется и содержание оригинального s в main
В нашем примере мы могли иначе объявить слайс и это повлияло бы на результат:
var s = make([]string, 3, 4)
s[0] = "1"
s[1] = "2"
s[2] = "3"
...
тогда при вызове функции поменялся бы и второй элемент и программа вывела бы [3 5 3], потому что при вызове append и добавлении четвертого элемента capacity бы еще хватало и память не перераспределялась - указатель бы сохранился таким, как и у оригинального s, а вот после второго append capacity уже бы не хватило, и память перераспределится, указатель у копии поменяется и как в первом случае не повлияет на оригинальный слайс
Почему в main не поменялся размер слайса? - потому что у копии поменялись length и capacity, а оригинал остался неизменно трехэлементным

Слайс представляет собой структуру, которую можно назвать оберткой для массива, эта структура содержит указатель на первый элемент некоторого массива, который содержит элементы слайса, длину и вместимость (length и capacity) этого слайса, размер отображает текущую длину (количество элементов), а вместимость - длину внутреннего массива. Когда появляется необходимость расширить слайс (когда capacity начинает не хватать), память выделяется на бОльшую вместимость и происходит копирование предыдущего массива (то есть, указатель будет ссылаться уже на другой участок памяти). Append при недостаточном capacity удваивает его (в нашем примере изначальный capacity в функции станет равным 6)

```
